# update_prd — синхронизация PRD с currentProblems

Обнови **`{{PRD_PATH}}`** по материалам из `docs/tasks/currentProblems.md`, чтобы требования были однозначными и готовыми к декомпозиции в задачи.

## Входные данные

1. `{{PRD_PATH}}`
2. `docs/tasks/currentProblems.md`
3. `docs/tasks/progress.md`
4. `docs/new-agents.md`

## Цель

Не переписать PRD с нуля, а аккуратно добавить/уточнить раздел текущих доработок (**`{{CURRENT_WORK_SECTION}}`** в `{{PRD_PATH}}`), сохранив связность с основными разделами PRD.

## Процесс

1. Прочитай currentProblems и выдели пункты, ещё не закрытые по смыслу в progress.
2. Для каждого пункта сформируй подпункт доработки:
   - контекст/проблема;
   - целевое поведение;
   - проверяемые acceptance criteria;
   - границы in-scope / out-of-scope;
   - связь с другими разделами PRD.
3. Если новая доработка меняет старое поведение, явно укажи, какая формулировка теперь канонична.
4. Обнови раздел «История изменений» в PRD.

## Стиль и ограничения

- Язык проекта (обычно русский для продуктовых формулировок).
- Пиши требования и поведение, не code-level инструкции.
- Нумерация подпунктов — монотонная и непротиворечивая.
- Не добавляй дубликаты уже реализованных требований.

## Выход

Кратко отчитай:

- какие пункты перенесены в PRD;
- какие пункты объединены/уточнены;
- что требует решения пользователя.

Следующий шаг: **`generate_tasks`**.
